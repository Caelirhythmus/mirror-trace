### 一、技术栈分析

| 层级 | 推荐技术 | 理由与关键点 |
| :--- | :--- | :--- |
| **渲染层** | **HTML5 Canvas 2D** | 无需引入重型WebGL，线条渲染适合双缓冲机制（左静态缓冲，右动态缓冲）。 |
| **输入层** | **Pointer Events API** | 统一处理鼠标、触摸板和**数位板**。务必监听 `pointerdown/move/up`，并通过 `pointer.getCoalescedEvents()` 获取高采样率点（数位板60Hz-240Hz输出），避免丢帧。 |
| **笔锋感应** | **W3C Pointer Events 扩展** | 读取 `pointer.pressure`（压力）和 `pointer.tiltX/Y`（倾斜角），在临摹时给予线条粗细/颜色深浅的实时视觉反馈。 |
| **核心算法层** | **纯原生 JavaScript / TypeScript** | 路径匹配涉及循环计算，无需第三方库，手写算法便于调参。 |
| **状态管理** | **Zustand 或 Vanilla JS** | 仅需管理 `isDrawing`、`currentPath`、`elapsedTime`，状态较轻量。 |

### 二、 随机线条生成策略（左侧画布）

线条具备“可临摹性”：

- **生成器**：采用 **3阶贝塞尔曲线（Cubic Bezier）** 拼接。
- **随机参数**：在画布坐标内随机生成 3~5 个控制点，确保曲线 **C1连续**（切线平滑）。
- **降采样输出**：将贝塞尔曲线按等距参数 `t`（步长 0.01）重采样为 **有序点数组** `P_origin = [{x, y}, ...]`，点数约 100~300 个。

---

### 三、 用户输入采集与预处理（右侧画布）

这是打好算法基础的关键：

1. **实时采集**：利用 `getCoalescedEvents` 收集物理层所有点，存入 `P_user`。
2. **轨迹压缩**：使用 **Ramer-Douglas-Peucker 算法** 过滤掉抖动冗余点（阈值 0.5px），保留特征点。
3. **重采样对齐**：**强制统一点数**。无论用户画多快多慢，使用 **等距插值（Linear Interpolation）** 将 `P_user` 重新采样为与 `P_origin` **完全相同的点数 N**。这是后续计算相似度的绝对前提。

---

### 四、 路径相似度打分算法（核心难点）

既然要求“路径相似度”+“消耗时间”，建议采用**混合加权评分法**（满分100分）：

#### 1. 空间相似度
单纯使用 **DTW（动态时间规整）** 在长路径上计算量过大且对点序漂移敏感。推荐组合算法：

- **百分位豪斯多夫距离**：忽略顶端 5% 的最远点，取剩余最大距离（40%）
- **普鲁克分析（仅平移变换）**（60%）

#### 2. 时间消耗分

- 设原始路径总长度为 `L`，设定最佳速度 `V_ideal = 80px/s`（可配置），得出理想时间 `T_ideal`。
- 分数采用负指数衰减模型（刚超一点就狂扣，速度越慢扣得越慢），以激励用户尽可能达到最佳速度。**只对过慢扣分**。

#### 3. 最终总分公式
`\text{FinalScore} = \big[(\text{ProcrustesScore} \times 0.6) + (\text{HausdorffScore}_{95\%} \times 0.4)\big] \times 0.65 + \text{TimeScore}_{\text{指数}} \times 0.35`。

### 六、 针对“数位板”的特殊优化

- **坐标偏移**：数位板驱动映射到浏览器时，务必使用 `getBoundingClientRect()` 计算相对坐标，避免 CSS 缩放导致偏移。
- **渲染性能**：用户绘画时，每帧只绘制 **增量线段**（`lineTo`），不要每帧 `clearRect` 重绘全部点，否则 CPU 占用极高。

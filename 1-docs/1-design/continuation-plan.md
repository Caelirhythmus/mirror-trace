## 阶段一：架构加固（推荐优先）

### 1.1 拆分 `main.ts`

当前 1463 行耦合了七块独立逻辑。拆分为独立模块：

| 目标文件 | 职责 | 估算移出行数 |
|---|---|---|
| `src/app.ts` | 生命周期、DOM 绑定、模式切换、键盘快捷键 | ~200 |
| `src/renderer.ts` | 参考画布绘制、热力图引导、十字线、分段高亮、`drawLineRanges` | ~250 |
| `src/history-manager.ts` | 笔画历史模型、undo/redo、`restoreStrokesUpTo`、`redrawUserCanvasContent` | ~250 |
| `src/input.ts` | 指针事件绑定、`setPointerCapture`、笔压处理、`clientToCanvas` | ~80 |
| `src/main.ts` | 缩减为入口，只保留应用状态 + 生命周期编排 | ~250 |

**拆分原则：**
- 每个模块只通过 public 方法暴露接口
- `main.ts` 只做依赖注入和事件桥接
- 拆分过程中不允许改现有行为

**风险：** 拆分过程中可能破坏现有交互流程——建议先用集成测试锁住行为再拆。

### 1.2 增加集成测试

现有 54 项测试只覆盖纯算法层（generator / trajectory / scoring / matching），`main.ts` 交互逻辑零覆盖。

**技术选型：** Vitest + happy-dom

**测试场景清单：**

```
模式切换
  ├─ 概括→单笔→多条→地狱 各模式切换后曲线类型正确
  └─ 地狱开关不污染 normal 模式状态

完整笔画流程
  ├─ pointerdown + move + up → 评分显示
  ├─ 单笔模式：一次 stroke 即完整评分
  ├─ 概括模式：多段 stroke 后触发全评价
  └─ 地狱复杂曲线：分段匹配 + 线级覆盖完成

Undo / Redo
  ├─ 各模式 undo 后画布状态正确
  ├─ 概括模式 undo 恢复覆盖进度
  └─ 跨模式 undo 边界安全

localStorage
  ├─ 单笔/多条/地狱模式存记录
  └─ 概括模式不存中间段，只存全评价

UI 可见性
  ├─ 地狱模式隐藏 mode-switch + 多条 toggle
  ├─ 概括模式显示复杂分段
  └─ 单笔模式显示多条 toggle
```

**收益：** 给拆分重构提供安全网，防止回归。

```

多线 3+2    → 直线 3、总量 5
```

## 阶段四：性能优化

| 问题 | 影响 | 修复方案 | 难度 |
|---|---|---|---|
| `hausdorff95` O(n²) | 评分时 400×400 = 160K 距离计算 | KD-Tree 或近似算法 | 中 |
| `rdpSimplify` 大量中间数组 | GC 压力 | 原地操作版本 | 低 |
| Canvas 每帧全量重绘 | 笔画中 CPU 占用偏高 | 脏矩形增量绘制 | 低 |
